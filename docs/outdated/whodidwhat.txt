
DOCUMENTATION INFORMATION FORM
FL - 14/11/2016
------------------------------

Goal :
As you might know, I need to document the whole Friend system, server, web-interface for both internal and client use.
Most of the source-codes of the project do not include the name of the authors and the name of the people who worked
after the initial author. I could find it with GIT, yet it would take a lot of manipulations, and I do not know any
solution that would allow me to do it quickly (if you know please tell me !)

This file is a Tree listing of the Friend Server content pulled today from GIT (14/11)
I have removed all the filenames that were not source-codes (like PNGs), and removed the external libraries to reduce
the number of entries.

I would like everyone of the developers to explore this file and in front of each file or directory -if he the one who
created all the content of this directory- to write his initials, so that a reference of who-made-what is available
for the company. At the end I will have a reference document that I can use to create a better doc for new
developers: they will easily know the person to contact for information, if the API doc -next on my schedule-
is not enough...

So I think it is worth spending a little time on this as the number of persons involved in the project may grow fast.
It will allow us to save a lot of time getting them fluent enough and be productive quicker.

It may be a little boring to do, yet I think this is really worth the time spend.

So please:
1. Set your text editor to REPLACE mode. Please respect the format of the file for the next
   person (put a bit of tape on BACKSPACE and DELETE in replace mode :)
2. Explore this file and for each folder, or each file eventually, write your initials in the "AUTHOR" column
   if you were the creator of the file / folder...
3. If you have modified some files of a folder of which you are NOT the author, write your initials in the
   "MODIFIED BY" column
4. Once done, send this file to the next person who will continue the task
5. Please put a note on the Documentation Trello card I have created when you pass this file to another person..
6. I will grab the file at the end, and fill-up the "ROLE" column discovering the sources one by one.
   It would help me too if you could, at the same time do this : a few words at the beginning of each section
   explaining the purpose of this section, only if the name of the files are not obvious enough.
7. It would be cool if from now on, when you create a new file, you would write your initials in the header!
8. People to work on this. The order is just a suggestion (apart from Hogne to be first)...
   HT

   EO
   BM

Thank you very much for your participation!
